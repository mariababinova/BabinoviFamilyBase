# Portal Patterns

## Home page pattern

Home page should have:

1. Sidebar with top-level modules.
2. Search bar.
3. Large brand title.
4. Short tagline.
5. Five portal cards.
6. Quick actions.

Do not include operational data on the home page unless explicitly requested.

## Module card pattern

Each module card should include:

- large circular glowing icon;
- module name;
- short description;
- small meta row;
- circular arrow action.

## Module overview page pattern

Module overview pages should be minimal.

Example: Health overview page should show:
- title;
- health profile cards;
- health quick actions.

Detailed data lives inside profiles.

## Anti-patterns

Avoid:

- all data on one screen;
- long feeds on portal pages;
- duplicate people list in sidebar and content;
- counters as main content;
- admin dashboard layouts;
- flat cards without icons;
- visual decisions without meaning.
